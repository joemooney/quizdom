# Reserved paths

AIDA planning prompts include a project-local reserved namespace list from
`.aida/reserved-paths.toml`. The list helps `/ultraplan` avoid proposing
files in directories already owned by AIDA generators, runtime state, agent
configuration, or workspace crates.

The baseline file is scaffolded by `aida init`. Add project-specific entries
when a path becomes owned by a tool or convention:

```toml
[[reservations]]
path = "docs/product/"
reason = "reserved for product-facing docs generated by the release process"
```

Keep reasons short and operational. The assembled `aida ultraplan <SPEC>`
prompt renders them under `## Reserved namespaces and conventions`, and
`aida ultraplan --json` exposes the same data as `reservations`.
